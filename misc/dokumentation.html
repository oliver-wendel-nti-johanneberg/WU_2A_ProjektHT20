Jag använde först position fixed vilket först inte fungerade då det tog bort scrolljulet på sidan av sidan. Jag testade sedan med position sticky men nav taggen följde inte med. Jag fick position fixed att fungera då jag specifierade position på top och left och som låg över headern.

4. Testning

Jag har svårt att få liveserver att fungera så jag vet ej om jag kommer kunna hinna med att titta på andra sidor. Jag hade kunnat undevika detta genom att strictare följa min planering samt inte snöa in på för advancerade tekniker inom webbutveckling. Med contrastchecker fick jag alla rätt på en del av hemsidan. Då mina sidor är så stora så kan jag inte testa hela sidan i contrastchecker. Jag använde också ett valideringsverktyg fick ett fel vilket var att jag hade använt samma id två gånger. Samt att jag inte hade underubriker på grid-blocken. Det är enkelt att fixa id men jag har aldrig tänkt på att använda underubriken inom css grid vilket jag får se inom senare projekt. Jag öppnade filerna utan liveserver med webbläsarna mozilla firefox, microsoft edge och google chrome. Alla dessa fungerade utan problem jag har bifogat bilder i misc mappen. De här verkar fungera bra dock kan äldre eller mindre webbläsare ha problem då de inte har lika mycket resurser och kanske inte samma standrarder. JAg tänker främst på tillexempel css grid som kom ut ganska nyligen och många webbläsare kan ha slutats uppdatera vilket skulle göra att gridsystemen skulle kollapsa.

5. Utvärdering (skrivs i slutet av arbetet, innan inlämning)

Jag tycker att det mesta har gått bra. Det som har gått mindre bra är strukturen i html och css. Har inte lärt mig än vilka klasser som skall återanvändas då jag märkte ganska snabbt att det var lätt att återanvända css och html kod över flera sidor igenom att tillexempel styla en klass istället för ett id. Dock vet jag ej vad man skall göra när man behöver två klasser och om man skall återanvända allt för mycket då det kan leda till stora error då man ändrar en sak på ett ställe som sedan ändrar tio sidor. Jag har också märkt att jag försöker att använda tekniker som är väldigt långt ifrån min kodningsförmåga som saktade ner arbetet mycket då det tog extremt lång tid att få sakerna att funka. Några förbättrigsförslag för denna sidan är att jag optimerar struktur och bilder då jag hade svårt att optimisera för bilder då jag återanvände många klasser. Jag skulle också ha fokuserat mer på dokumentation då vi betygsätts på det och inte på advancerade tekniker(dock är det väldigt roligt med advancerade tekniker). Har bifogat bilder i separat fil då jag inte hade tid att koda in de med hjälp av img taggar.
